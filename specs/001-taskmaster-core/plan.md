## Summary

TaskMaster Pro is a complete task management web application enabling users to create, view, filter, edit, and delete tasks with automated reminders. The application uses a dual-container architecture deployed on Azure Container Apps: an Angular 18 frontend with TailwindCSS for mobile-first responsive design, and a Python FastAPI backend with Pydantic validation. Data is persisted in Azure Database for MySQL Flexible Server. The system implements strict API contracts (GET/POST/PUT/DELETE /api/tasks) with 80% minimum test coverage using pytest (backend) and Jest (frontend). All code is generated with GitHub Copilot assistance following TDD principles.

## Technical Context

**Language/Version**: 
- Frontend: TypeScript 5.x (with Angular 18)
- Backend: Python 3.11+

**Primary Dependencies**: 
- Frontend: Angular 18, TailwindCSS, RxJS, Angular HttpClient
- Backend: FastAPI, Pydantic, SQLAlchemy, Alembic (migrations), mysql-connector-python

**Storage**: Azure Database for MySQL Flexible Server

**Testing**: 
- Frontend: Jest (unit tests), Angular Testing Library
- Backend: pytest, pytest-cov (coverage), pytest-asyncio

**Target Platform**: 
- Deployment: Azure Container Apps (dual containers: frontend + backend)
- Client: Modern web browsers (Chrome, Firefox, Safari, Edge - latest 2 versions)
- Runtime: Docker containers orchestrated by Azure Container Apps

**Project Type**: Web application (full-stack SPA with RESTful API backend)

**Performance Goals**: 
- Dashboard load time: <2 seconds for up to 1000 tasks
- API response time: <500ms p95
- Mobile performance: No layout shift, <3s first contentful paint on mobile networks

**Constraints**: 
- Mobile-first design: Minimum 375px viewport width support
- Touch targets: Minimum 44×44px for accessibility
- Test coverage: Minimum 80% for both frontend and backend
- API contract: Immutable endpoints without versioning (GET/POST/PUT/DELETE /api/tasks)
- Azure-native only: No hybrid cloud or non-Azure services

**Scale/Scope**: 
- Expected users: Small to medium teams (10-100 concurrent users)
- Task volume: Up to 10,000 tasks per user efficiently supported
- Single-entity model: Task entity only (no user assignments, teams, or complex relationships in v1)
